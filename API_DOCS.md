# وثيقة واجهة برمجة التطبيقات للمصادقة والمستخدمين (API Auth & Registration Docs)
## مشروع إدارة المدارس (النقل والمخزون)

تهدف هذه الوثيقة المرجعية المخصصة إلى تزويد مطور الفرونت-إند بالبيانات والتعليمات اللازمة لربط عمليات **المصادقة (Authentication)، وإنشاء الحسابات (Registration)، واستعادة كلمة المرور** فقط.

---

### إرشادات الربط العامة والمشتركة (General Guidelines)

#### 1. الـ Headers المطلوبة في كل الطلبات
يجب على مطور الفرونت-إند إرفاق هذه الـ Headers في كل طلب يرسله إلى الخادم:

| الـ Header | القيمة | الوصف |
| :--- | :--- | :--- |
| `Accept` | `application/json` | إجباري لإخبار السيرفر بإرجاع الرد بصيغة JSON دائماً وتفعيل معالجة الأخطاء الافتراضية. |
| `Accept-Language` | `ar` | إرسال القيمة لضمان الحصول على رسائل التحقق وأخطاء البيانات باللغة العربية الفصحى. |
| `Authorization` | `Bearer {TOKEN}` | إجباري في المسارات المحمية بالتوكن (مثل تسجيل الخروج وإضافة مشرف). |

#### 2. تدفق المصادقة وحفظ التوكن (Auth Token Flow)
- بعد تسجيل الدخول بنجاح عبر مسار `/api/auth/login` أو التسجيل الناجح لولي الأمر، سيرجع الباك إند قيمة التوكن في حقل `access_token` ونوع التوكن `token_type` كـ `Bearer`.
- يجب على الفرونت-إند تخزين هذا التوكن بأمان داخل `localStorage` أو في `Cookies` وإرفاقه تلقائياً داخل حقل الـ `Authorization` كـ `Bearer {token}` في كل طلب لاحق للمسارات المحمية.

#### 3. معالجة رفع الملفات والصور (Media & File Handling)
- عند التعامل مع مسارات رفع الصور والملفات (مثل: الصورة الرمزية للسائق `avatar_url` أو صورة المشرف `avatar_url`)، يجب إرسال الطلب بنوع محتوى `multipart/form-data` وليس `application/json`.

---

### أكواد الاستجابة الشائعة (Common Response Codes)

| كود الحالة | الاسم التقني | الوصف البرمجي في النظام |
| :--- | :--- | :--- |
| **`200`** | OK | تم تنفيذ الطلب بنجاح. |
| **`201`** | Created | تم إنشاء الحساب أو السجل بنجاح في قاعدة البيانات. |
| **`400`** | Bad Request | فشل الطلب بسبب خطأ منطقي (مثل رمز تحقق OTP منتهي أو خاطئ). |
| **`401`** | Unauthorized | التوكن مفقود أو منتهي الصلاحية. |
| **`422`** | Unprocessable Content | فشل التحقق من البيانات المرسلة (يرجى تصحيح الحقول الخاطئة). |
| **`500`** | Internal Server Error | خطأ غير متوقع في الخادم. |

### وثائق المسارات المطلوبة (Endpoints Reference)

#### 1. تسجيل الدخول وتسجيل الخروج للجميع (Login & Logout)

##### **تسجيل الدخول: `POST /api/auth/login`**
- **الوصف**: تسجيل الدخول للنظام لجميع الأدوار (سوبر أدمن، مشرف، سائق، ولي أمر) والحصول على التوكن.
- **الحقول الإجبارية**:
  - `phone_number` (string): رقم هاتف يبدأ بـ 09 ويتكون من 10 أرقام (مثال: `0912345678`).
  - `password` (string): كلمة المرور.
  - `platform` (string): نوع المنصة، يقبل فقط: `ios` أو `android` أو `web`.
- **الحقول الاختيارية**:
  - `device_name` (string, nullable): اسم جهاز العميل (مثل: iPhone 14).
  - `fcm_token` (string): التوكن الخاص بالإشعارات (Firebase Cloud Messaging).
- **مثال على الرد الناجح (200 OK)**:
```json
{
    "status": true,
    "message": "تم تسجيل الدخول بنجاح!",
    "access_token": "1|sanctum_token_hash_here...",
    "token_type": "Bearer",
    "user": {
        "id": 5,
        "full_name": "أحمد محمد علي",
        "phone_number": "0912345678",
        "role_id": 3,
        "is_active": true
    }
}
```

##### **تسجيل الخروج: `POST /api/auth/logout`**
- **الحماية**: يتطلب (Bearer Token)
- **الوصف**: تسجيل الخروج وإبطال توكن الجلسة النشطة.
- **مثال على الرد الناجح (200 OK)**:
```json
{
    "status": true,
    "message": "تم تسجيل الخروج بنجاح."
}
```

#### 2. استعادة كلمة المرور للجميع (Password Recovery)

##### **أ. طلب إرسال رمز التحقق (OTP): `POST /api/auth/password/send-otp`**
- **الوصف**: إرسال كود تحقق بريد إلكتروني مسجل مسبقاً لاستعادة الحساب.
- **الحقول الإجبارية**:
  - `email` (string): البريد الإلكتروني للحساب.
- **مثال على الرد الناجح (200 OK)**:
```json
{
    "status": true,
    "message": "تم إرسال رمز التحقق إلى بريدك الإلكتروني."
}
```

##### **ب. إعادة تعيين كلمة المرور الجديدة: `POST /api/auth/password/reset`**
- **الوصف**: تعيين كلمة المرور الجديدة بعد التحقق من كود الـ OTP.
- **الحقول الإجبارية**:
  - `email` (string): البريد الإلكتروني.
  - `code` (string): كود التحقق المكون من 6 أرقام المستلم بالبريد.
  - `password` (string): الرقم السري الجديد (6 خانات كحد أدنى).
  - `password_confirmation` (string): تأكيد الرقم السري الجديد ومطابقته لـ password.
- **مثال على الرد الناجح (200 OK)**:
```json
{
    "status": true,
    "message": "تم تحديث كلمة المرور بنجاح."
}
```

#### 3. إنشاء حساب ولي أمر جديد (Parent Registration)

##### **أ. طلب إرسال رمز OTP للتسجيل: `POST /api/parent/send-otp`**
- **الوصف**: إرسال كود تحقق لبريد ولي الأمر للتأكد من صلاحيته قبل إتمام الحساب.
- **الحقول الإجبارية**:
  - `email` (string): البريد الإلكتروني (أقصى 100 حرف).
- **مثال على الرد الناجح (200 OK)**:
```json
{
    "status": true,
    "message": "تم إرسال كود التحقق بنجاح."
}
```

##### **ب. إتمام التسجيل النهائي لولي الأمر: `POST /api/parent/register`**
- **الوصف**: إنشاء حساب ولي الأمر بالتحقق من الرمز المدخل.
- **الحقول الإجبارية**:
  - `full_name` (string): الاسم الكامل ثلاثي (3 كلمات على الأقل تفصل بينها مسافات).
  - `email` (string): البريد الإلكتروني (يجب أن يكون فريداً وغير مسجل مسبقاً).
  - `phone_number` (string): رقم الهاتف الأساسي (7 خانات كحد أدنى، فريد).
  - `password` (string): كلمة المرور (7 خانات كحد أدنى، يجب أن تحتوى على حروف وأرقام، ويُمنع استخدام الرموز الخاصة).
  - `password_confirmation` (string): تأكيد كلمة المرور ومطابقتها لـ password.
  - `otp` (string): رمز التحقق المستلم (6 أرقام).
- **الحقول الاختيارية**:
  - `alternative_phone` (string): رقم هاتف بديل للطوارئ (7 خانات كحد أدنى).
- **مثال على الرد الناجح (201 Created)**:
```json
{
    "status": true,
    "message": "تم إنشاء الحساب بنجاح.",
    "data": {
        "id": 1,
        "full_name": "أحمد عبد الله الترهوني",
        "email": "ahmed@example.com",
        "phone_number": "0923456789",
        "role": "parent"
    },
    "token": "2|parent_token_hash_here..."
}
```

#### 4. إنشاء حساب سائق جديد (Driver Registration)

##### **أ. إنشاء الحساب (المرحلة الأولى): `POST /api/v1/driver/register`**
*ملاحظة: يدعم رفع الصورة الرمزية لذا يجب أن يكون نوع المحتوى `multipart/form-data`.*
- **الحقول الإجبارية**:
  - `full_name` (string): الاسم الكامل ثلاثي (بين 10 إلى 100 حرف).
  - `email` (string): البريد الإلكتروني للسائق (فريد).
  - `phone_number` (string): رقم الهاتف (10 أرقام، فريد، ويبدأ بـ 09).
  - `gender` (string): الجنس، يقبل فقط: `male` (ذكر) أو `female` (أنثى).
  - `password` (string): كلمة المرور (6 خانات كحد أدنى).
- **الحقول الاختيارية**:
  - `avatar_url` (file): صورة شخصية للسائق (ملف صورة jpeg, png, jpg، أقصى 2 ميجابايت).
- **مثال على الرد الناجح (201 Created)**:
```json
{
    "status": true,
    "message": "تم إنشاء الحساب بنجاح، يرجى تفعيل الرمز المرسل لإيميلك.",
    "user_id": 12
}
```

##### **ب. تفعيل وتأكيد حساب السائق: `POST /api/v1/driver/verify-otp`**
- **الوصف**: التحقق من رمز OTP لتنشيط تفعيل الحساب.
- **الحقول الإجبارية**:
  - `email` (string): البريد الإلكتروني للسائق.
  - `otp` (string): رمز التحقق المكون من 6 أرقام.
- **مثال على الرد الناجح (200 OK)**:
```json
{
    "status": true,
    "message": "تم التحقق من الرمز بنجاح وتفعيل الحساب."
}
```

##### **ج. إكمال الملف المهني والسيارة والمستندات (المرحلة الثانية): `POST /api/v1/driver/complete-profile/{userId}`**
- **الوصف**: إرسال المستندات والبيانات المهنية للسائق والسيارة للمراجعة.
- **الحقول الإجبارية**:
  - `national_id` (string): الرقم الوطني للسائق (بين 10 و 20 حرف).
  - `license_number` (string): رقم رخصة القيادة (أقصى 50 حرف).
  - `license_expiry` (date): تاريخ انتهاء رخصة القيادة في المستقبل.
  - `plate_number` (string): رقم لوحة السيارة (أقصى 20 حرف).
  - `brand` (string): ماركة المركبة (أقصى 50 حرف).
  - `model` (string): موديل المركبة (أقصى 50 حرف).
  - `year` (integer): سنة صنع المركبة (بين 1990 وحتى العام القادم).
  - `color` (string): لون المركبة (أقصى 30 حرف).
  - `type` (string): نوع المركبة (مثال: Bus, Sedan, Van - أقصى 30 حرف).
  - `capacity_manual` (integer): عدد المقاعد المتاحة للطلاب (بين 1 و 20 مقعد).
  - `has_ac` (boolean): توفر مكيف هواء بالسيارة (0 أو 1).
  - `vehicle_image_path` (string): الرابط النصي لصورة السيارة المرفوعة مسبقاً.
  - `doc_license_path` (string): الرابط النصي لصورة رخصة القيادة المرفوعة مسبقاً.
  - `doc_logbook_path` (string): الرابط النصي لصورة كتيب المركبة المرفوع مسبقاً.
  - `doc_insurance_path` (string): الرابط النصي لصورة تأمين المركبة المرفوع مسبقاً.
  - `doc_criminal_record_path` (string): الرابط النصي لشهادة السجل الجنائي المرفوعة مسبقاً.
- **مثال على الرد الناجح (200 OK)**:
```json
{
    "status": true,
    "message": "تم رفع البيانات بنجاح، بانتظار مراجعة الإدارة.",
    "data": {
        "id": 4,
        "driver_status": "Pending",
        ...
    }
}
```

#### 5. إضافة مشرف جديد (Add Supervisor)

##### **إضافة مشرف جديد: `POST /api/admin/admins`**
- **الحماية**: يتطلب (Bearer Token + Admin Role)
- **الوصف**: قيام السوبر أدمن بإضافة مشرف (أدمن فرعي) جديد للنظام.
- **الحقول الإجبارية**:
  - `full_name` (string): الاسم الكامل ثلاثي باللغة العربية (فريد، مثال: خالد عبد الله محمود).
  - `email` (string): البريد الإلكتروني للمشرف (فريد).
  - `phone_number` (string): رقم الهاتف (10 أرقام، فريد، ويبدأ بـ 09).
- **الحقول الاختيارية**:
  - `password` (string, nullable): كلمة المرور (إذا تركت فارغة سيقوم السيرفر بتوليد باسورد قوي عشوائي وتلقائي).
  - `avatar_url` (file): صورة رمزية للمشرف (ملف صورة jpeg, png, jpg، أقصى 2 ميجابايت، يتطلب نوع محتوى `multipart/form-data`).
- **مثال على الرد الناجح (201 Created)**:
```json
{
    "status": true,
    "message": "تم إضافة المشرف بنجاح.",
    "data": {
        "id": 3,
        "full_name": "خالد عبد الله محمود",
        "email": "khaled@example.com",
        "phone_number": "0912222222",
        "avatar_url": null,
        "is_active": true
    }
}
```
